Splash Screen

Technical Description

The Splash Screen serves as an engaging entry point. Structurally, it uses a full-screen layout with absolute positioning to layer a gradient background, decorative CSS-animated bubbles, and a centered content container. I utilized CSS @keyframes for the rising bubbles, the bobbing ocean icon, and the loader bar animation to create a dynamic visual experience without relying on external video files. Functionally, JavaScript is used to manage a countdown timer via setTimeout(), updating the DOM dynamically. Once the countdown reaches zero, or if the user clicks the "Skip Intro" button, an automatic redirect to the Home page is triggered via window.location.href.

Accessibility

To ensure accessibility, the main content area is wrapped in a div with role="region" and an aria-label. Decorative elements like the background and animated bubbles are hidden from screen readers using aria-hidden="true". The countdown timer utilizes aria-live="polite" so that screen readers announce the changing text naturally. The loader bar also includes standard progress bar ARIA attributes (role="progressbar", aria-valuemin, aria-valuemax).

Action Impact Simulator (AIS) Page

Technical Description

This page uses an embedded CSS layout featuring CSS Grid (grid-template-columns: repeat(auto-fit, minmax(260px, 1fr))) to create a responsive layout for the action cards. The interactivity is entirely JavaScript-driven. When a user clicks an action card, a JS function toggles its selection state, storing the selection in an object. The script then calculates the total points, dynamically updates the impact score in the DOM, adjusts the width of the visual progress bar, and categorizes the impact level (Low, Medium, High). Based on this level, JS adds specific classes to the background element (#ais-page-bg) to smoothly transition the page's color scheme.

Accessibility

The action cards are fully keyboard accessible; they feature a tabindex="0" and listen for both 'Enter' and 'Space' key events to toggle their state. The cards utilize role="checkbox" and dynamically update their aria-checked status via JavaScript to inform assistive technologies. The impact result panel features aria-live="polite" so users are immediately notified when their score and impact level change.

Content Page: Marine Life in Danger

Technical Description

The Content Page is structured using semantic HTML5 tags including <main>, <h1> to <h3>, and <ul> to establish a clear hierarchy. To manage the long-form content, I implemented internal navigation using anchor links (href="#section-id") that correspond to specific sections on the page. The layout heavily relies on CSS Flexbox for the top statistics bar and CSS Grid for the endangered species cards, ensuring a fluid, responsive design. A "Go to Top" button is implemented using CSS position: fixed so it remains accessible in the bottom corner as the user scrolls.

Accessibility

Accessibility is maintained by ensuring a logical heading hierarchy, which aids navigation for screen reader users. All images—whether decorative or informative—include descriptive alt text (e.g., "Bleached white coral reef with very few fish remaining"). High contrast text colors were selected to ensure readability against the background gradients, and the internal navigation menu features an aria-label.

Challenges and Lessons Learned

One major challenge was implementing the dynamic background color changes on the AIS page without causing jagged transitions or relying solely on CSS :hover states. I resolved this by decoupling the background from the document body, placing it in a fixed full-screen div (#ais-page-bg) with a CSS transition, and using JavaScript to add or remove class names (low, medium, high) based on the calculated score. This taught me how to effectively bridge JavaScript logic with CSS state changes. Another challenge was making custom div elements behave like checkboxes for accessibility; I learned how crucial it is to pair tabindex with explicit keyboard event listeners to replicate native HTML input behavior.

Compliance

The pages I have implemented comply with JANET regulations governing web page publication. All content is strictly educational and directly aligned with the UN's Sustainable Development Goal 14 (Life Below Water). The site does not host commercial advertising, offensive material, or unauthorized software. Furthermore, academic integrity is maintained: all third-party placeholder images are sourced from royalty-free platforms (like Picsum), and the written content was independently researched and authored rather than copied directly from UN sources.
