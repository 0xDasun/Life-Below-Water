Home Page

Technical Description

The Home Page acts as the central entry point and uses the shared HTML template to maintain visual consistency. A full-screen background video was implemented using the <video> tag with autoplay, muted, and loop attributes, positioned fixed behind the content via CSS. The main content is structured using semantic tags like <section> and <article>. To display the mission cards, a Flexbox layout (.cards-container) was utilized to ensure the cards wrap properly on smaller screens. The interactive gallery links are structured using CSS Grid (.gallery-container) with responsive columns (auto-fit and minmax). Embedded CSS includes transition properties to create smooth hover effects (transform and background-color changes) for the gallery items.

Accessibility

Accessibility is addressed by using semantic HTML structures which assist screen readers in parsing the page. All images within the mission cards include descriptive alt text (e.g., "Plastic pollution floating in the ocean"). The background video is muted by default to prevent unexpected audio playback, which is a key accessibility standard. High contrast text colors were chosen for readability over the transparent main content container.

Gallery Page

Technical Description

The Gallery Page features a dynamic layout using CSS Grid (.thumbnail-grid) to perfectly align the thumbnail cards. JavaScript is heavily utilized to handle the interactivity, separating the data structure (an array of image objects) from the DOM rendering logic. Event listeners are applied to the generated cards to trigger a modal overlay (#extendedView) when clicked. The modal displays the full-sized image and description dynamically. Furthermore, JavaScript is used to implement user customization features: a toggle for Dark Mode and a button to switch the font style between Georgia and Courier New, directly manipulating the DOM style properties of the modalBox.

Accessibility

Accessibility was incorporated by ensuring the modal overlay can be closed not only via the close button but also by clicking outside the content box, providing a larger click target for users. Descriptive alt text is dynamically injected into the modal image based on the selected thumbnail. The hover effects on the thumbnails provide visual feedback to users that the elements are interactive.

Content Page (The State of Our Oceans)

Technical Description

This page delivers structured informational content using a logical hierarchy of headings (<h1> to <h2>). Internal page navigation is facilitated through an unordered list where anchor tags (<a href="#id">) link directly to specific section IDs on the page. A CSS Grid is used for the .content-section layout to align images and text side-by-side cleanly. Flexbox is employed for the .stat-bar to distribute statistics evenly. A persistent "Go to top" button is implemented using CSS position: fixed at the bottom right, allowing quick navigation back to the header without relying on JavaScript.

Accessibility

The internal navigation menu utilizes the aria-label="Page sections" attribute to clarify its purpose to assistive technologies. The "Go to top" link is similarly equipped with an aria-label. All informational and decorative images include detailed alt text describing the scene (e.g., "Aerial view of a clear blue ocean stretching to the horizon"). Semantic emphasis tags like <strong> are used appropriately rather than relying purely on CSS font-weights.

Challenges and Lessons Learned

One of the primary challenges was ensuring the JavaScript logic for the Gallery modal worked seamlessly while maintaining separation of concerns. Initially, getting the dynamically generated thumbnails to trigger the correct modal data required careful event listener binding. I learned the value of storing data in an array of objects and rendering the DOM dynamically, which keeps the HTML file clean and makes the gallery easily scalable. Another challenge was implementing the shared navigation template across all pages while keeping the "active" class correct for each page; this reinforced the importance of careful file structuring and CSS class management.

Compliance

The pages I developed adhere to the JANET regulations governing web page publication. External resources, such as images, have been appropriately sourced (e.g., royalty-free or educational contexts) to comply with copyright regulations and ensure academic integrity.
