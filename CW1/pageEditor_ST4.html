User Profile Page

Technical Description

This page uses a combination of HTML, CSS, and JavaScript to create an interactive form. A CSS-styled progress bar visually tracks completion using a transition property for smooth animation. The logic relies on JavaScript prompt methods to capture user data, which is stored in a data object. I used document object model manipulation combined with if and else statements to dynamically display or hide specific output containers based on whether the user provided data.

Accessibility

I used semantic HTML headings to establish a clear document outline. The button elements have clear, descriptive text explaining each step, and the output area uses high-contrast text against a solid background for maximum readability.

Site Map Page

Technical Description

I implemented a visual, interactive sitemap using an inline scalable vector graphics element. The structure is built using basic shapes like rectangles, lines, and text to map out the website's hierarchy. For interactivity, I wrapped the graphic nodes in hyperlink elements and used CSS styling rules combined with transition properties to change the border and fill colors of the shapes when a user hovers over or clicks them.

Accessibility

To ensure the graphic is accessible, I assigned an image role and a descriptive aria label to the main container. Every clickable node is configured to allow full keyboard navigation using the tab key. I also included specific title elements inside the links so screen readers can clearly announce the destination of each block to visually impaired users.

Content Page

Technical Description

This page uses a CSS Grid layout to display images alongside text in a responsive, two-column format. I created an alternating visual rhythm by applying a specific styling class to swap the text and image directions as the user scrolls down. The design features CSS linear gradient backgrounds for the hero section and navigation, alongside a 'Go to Top' button that's at the end of the page.

Accessibility

I ensured all marine technology images have descriptive alternative text so their content can be understood without seeing the screen. I used proper semantic navigation elements for the internal page menu to establish a clear document outline. For the 'Go to Top' link centered at the bottom of the page, I made sure the button has a large, easily clickable area and uses high-contrast, readable text. This ensures it is fully accessible and easy to interact with for users who may have motor impairments or rely on keyboard navigation.

Challenges and Lessons Learned

One major challenge I faced was implementing the alternating layout on the Future Conservation Tech page using CSS Grid. At first, I tried to create completely separate HTML structures for the left-aligned and right-aligned rows. It made my code incredibly messy and hard to maintain. I eventually researched and learned how to use the CSS direction property in combination with a 'reverse' class. This allowed me to keep the HTML structure identical for every section while letting CSS handle the visual flipping, which taught me a huge lesson in writing clean, efficient code.
Another big challenge was creating the interactive sitemap. Getting the clickable links to work smoothly within a vector graphic required a lot of testing. I had to learn how to properly wrap vector shapes in hyperlink elements and apply CSS transitions directly to the SVG shapes. Overcoming this significantly improved my understanding of how to build interactive, non-standard web elements.

Compliance

The pages I developed strictly adhere to the Janet Acceptable Use Policy, which governs web publication at the University of Westminster. The website is entirely educational, designed to raise awareness for Sustainable Development Goal 14: Life Below Water, and it does not contain any commercial advertising, offensive material, or discriminatory content. To respect copyright laws, all images used on my Future Conservation Tech page are sourced from royalty-free platforms. Furthermore, the User Profile page respects data privacy; it only uses temporary local variables to simulate a profile and does not permanently store, transmit, or harvest any personal data from the user.
